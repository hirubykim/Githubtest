원본 저장소를 복사해서 새로운 원격저장소를 만든다. = 포크

A개발자는 원본 저장소에 바로 커밋을 올릴 권한이 없습니다. 대신 A개발자는 원본 저장소를 복사해서 본인의 깃허브에 
새로운 원격 저장소를 만들고 이곳에 커밋을 올릴 예정입니다. 새로운 원격 저장소는 A개발자만 사용하는 곳이므로 
이곳에서 온갖 실험적인 커밋을 올릴 수 있습니다. 이렇게 남의 원본 저장소를 내 계정의 원격 저장소로 복사해오는 명령어를 포크라고 합니다.

기본적으로 원본 저장소에서 커밋을 직접 푸시할 수 있는 사람은 원본저장소를 만든 본인뿐입니다.
다른 사람이 이곳에 푸시하려면 원본 저장소의 소유자가 이 사람을 협력자로 등록해야 합니다 등록되지 않은 사람은 이곳에 푸시하면 에러가 납니다.
협력자로 등록하는 방법은 깃헙 사이트에서 setting - collaborators 페이지에 들어가서 등록하려는 유저를 찾고 add collaborators 를 클릭하면 됩니다.

그런데 원본 저장소의 소유자는 협력자가 많으면 원본 저장소를 관리하기 어렵습니다.
협력자가 원복 저장소에 직접 푸시할 수 있기 때문입니다. 하지만 소유자가 동시에 많은 개발자에게 의견을 받고 싶은 요구가 있고 
다른 개발자 역시 오픈 소스를 개선하고 싶은 요구가 있을 수 있습니다.
(그렇지만 원본에 직접 푸시하는 것에 대한 부담은 있습니다)

이럴 때 대안이 될 수 있는 방법이 풀 리퀘스트입니다. 
개발자는 원본 저장소를 자신의 계정에 복사(fork)해서 원격저장소를 생성하고 이곳에 커밋을 올린 후 원본 저장소의 소유자에게 병합을 요청하면(pull request)
원본 저장소의 소유자는 개발자의 병합 요청을 검토해서 원본 저장소에 반영하는 것입니다.

브랜치를 통해 코드 분기점을 만들고 풀 리퀘스트를 통해 서로 확인하고 병합하는 과정이랑 비슷합니다.
다만, 포크한 원격 저장소의 브랜치에서 원본 저장소의 브랜치로 풀 리퀘스트를 보내는 것입니다.

브랜치가 원격저장소 내에서 다른 코드를 만드는 것이라면 포크는 원격 저장소 밖에서 다른 코드를 만드는 것입니다.
포크는 브랜치를 포함한 원본 저장소의 모든 커밋 이력을 새로운 원격 저장소로 통째로 복사합니다. 

브랜치에 올린 커밋 이력은 원본 저장소에서 바로 볼 수 있는 것에 반해 복제한 원격 저장소는 새로운 주소로 독립되었기 때문에
이곳의 이력을 보려면 추가적으로 이곳의 원격저장소의 주소를 등록해야 합니다.

두 방법 모두 협업을 위해 만들어진 방법이므로 상황에 따라 알맞는 방법을 취사 선택해야 합니다.
소수의 개발자로 협업하는 경우는 브랜치를 나누는 것이 더 효율적이고 다수의 개발자로 협업하는 경우는 fork를 활용하는 게 효율적입니다.

브랜치
의의: 하나의 원본 저장소에서 분기를 나눈다. 
장점: 하나의 원본 저장소에서 코드 커밋 이력을 편하게 볼 수 있다.
단점: 다수의 사용자가 다수의 브랜치를 만들면 관리하기 어렵다.

포크
의의: 여러 원격저장소를 만들어 분기를 나눈다.
장점: 원본 저장소에 영향을 미치지 않으므로 원격 저장소에서 마음껏 코드를 수정할 수 있다.
단점: 복제한 원격 저장소의 이력을 볼려면 따로 주소를 추가해야 한다.

하나의 메일주소를 여러개처럼 사용하기
gmail의 + 기능을 사용하면 이메일 주소 하나로 마치 여러개의 이메일 주소인 것처럼 사용할 수 있습니다.
만약 test@gmail.com 주소를 가지고 있으면 test+sub@gmail.com 혹은 test+1@gmail.com 처럼 +와 함께 
뒤에 아무 글자를 붙여 가입해도 모든 메일을 test@gmail.com 으로 받을 수 있습니다.

포크하는 방법
1. 원본 저장소의 주소로 들어갑니다.
2. 우측상단에 fork 버튼을 누릅니다. 
3. 내 계정에 해당 원본 저장소가 복사됩니다.
4. 복사가 완료가 되면 새로운 주소의 원격저장소로 페이지가 이동하고 원격저장소 밑에 forked from  ~~ 이라는 설명이 달림

포크가 성공적으로 처리되면 이제 소유자가 본인이므로 마음대로 커밋을 올리거나 브랜치를 만들 수 있습니다.

이제 포크한 깃을 소스트리에 가져오는 방법은 이전에 했던 방법을 참고해서 가져오면 된다.
깃허브 "clone or download" 버튼 클릭 > 소스트리 새탭에서 clone 버튼 누르고 url 주소 입력 > 원격저장소와 연결할 로컬저장소 폴더 만들어서 입력 > 이름 설정 > 클론 버튼 누름

이후에 가장 먼저할 것은 새로운 계정으로 소스트리에 새로 로그인하는 것이다
당연하게 다른 계정으로 원격저장소에 푸시를 하게 되면 오류 메시지가 뜬다.

도구 - 옵션 - 인증 - 호스팅 서비스는 github로 선호 프로토콜은 https로 인증은 OAuth 혹은 Basic (둘 중 아무거나)
Basic은 닉네임과 암호를 입력해서 로그인하는 방식이고 OAuth는 웹 브라우저를 띄워서 로그인하는 방식이다. 
Basic으로 실습 진행 > 비밀번호 새로고침 클릭 후 비밀번호 인증 > 인증성공이 뜨면 확인 버튼 누름

"설정 초기화" 버튼을 누르면 해당 계정이 소스트리의 기본 계정이 됩니다.

이제 포크한 내용을 본인 마음대로 수정하고 커밋해서 본인의 계정에 반영할 수 있습니다.
소스트리에서 커밋할 때 코멘트 칸에 사람 모양 아이콘을 클릭하면 대체작성자 정보를 수정할 수 있다.
