소스트리에서 깃을 사용하는 방법

소스트리를 통해 깃을 커밋했으면 
로컬 저장소 버전에서는 master
원격 저장소 버전에서는 origin/master 로 표기될 거임

origin은 우리가 연결한 github 원격저장소의 닉네임
로컬저장소에서 원격저장소의 주소를 알려주기 위해 
$git remote add origin (깃허브 주소) 명령어를 수행했는데
이때 origin이란 이름으로 원격저장소를 추가하라는 의미였음
즉, 원격저장소의 닉네임인 것이다.

암튼 커밋을 한 상태면 원격저장소를 가리키는 버전과 로컬저장소를 가리키는 버전이 보일 것이다.

master는 사용자가 커밋을 올리는 메인 줄기이다. 커밋이 줄줄이 하나의 줄기로 이어져있는 것을 볼 수 있다. 
이는 따로 branch를 생성하지 않았기 때문이다. 브랜치를 생성하지 않으면 git은 mater라는 기본 줄기에 커밋을 올린다.
일반적으로 회사에서 master는 회사 프로그램을 최종적으로 상업에 내놓는 결과물 branch이다. 
그래서 실습에서는 바로 master라는 줄기에 바로 커밋을 하고 push를 진행하지만 이는 현업에서 적절하지 못하다.

종합해보면 아무것도 붙지 않은 master는 내 컴퓨터 로컬 저장소의 버전을, 앞에 origin이 붙은 origin/master는 github의
원격저장소의 버전을 가리키는 것입니다.
