원본 저장소를 본인 계정으로 통째로 복사(포크)한 뒤에 커밋을 올리고 원본 저장소에 반영하는 방법을 알아보자

포크한 원격저장소에서 원본저장소로 풀리퀘스트 보내기
앞서 풀리퀘스트를 배운 적이 있습니다. 
'A브랜치에서 B브랜치를 합치려고 하는데 새로 추가된 코드는 다음과 같다는 형식의 메시지를 보내는 것'이라고 배웠습니다.

포크한 원격 저장소에서 원본 저장소로 코드를 합치는 과정도 똑같습니다. 
더 정확히 말하면 포크한 원격저장소의 A브랜치에서 원본저장소의 B브랜치로 코드를 합치는 것입니다.

포크한 원격 저장소에서 풀리퀘스트를 보내는 방법
1. 포크한 원격 저장소 계정에 들어간다.
2. "NEW pull request" 버튼을 누르면 풀리퀘스트를 보내기 전 정보를 보여주는 화면이 보입니다.
3. Head repository에서 내가 포크한 원격 저장소가 base repository에 원본 저장소가 보여지면 성공입니다.
(화살표 방향을 보시면 이해하는데 도움이 되실 겁니다. head의 변경사항을 base에 합치려 한다는 의미입니다)
4. 아래에 able to merge라고 쓰여진 텍스트가 보이면 충돌 없이 바로 병합이 가능하다는 의미입니다.
5. 모든 정보가 확실하면 create pull request 버튼을 누릅니다. 
(버튼을 누르기 전 아래의 변경파일 changed files 정보를 꼼꼼하게 확인해보세요)
6. 풀리퀘스트에 대한 설명을 적고 디자인 변경사항은 캡쳐해서 첨부하는 것도 좋습니다.

참고. 오픈소스에 풀리퀘스트를 보내기전에 컨트리뷰션 가이드라인이 있는 지 확인하는 것이 좋습니다.
보통 원본 저장소의 readme.md 파일에 링크되어 있습니다.

7. 성공적으로 풀리퀘스트를 보내면 원본 저장소의 소유자의 풀 리퀘스트 승인과 병합을 기다리면 됩니다.


풀 리퀘스트 승인하고 병합하기
원본 저장소의 소유자 계정으로 로그인하면 누가 포크했는지 알 수 있습니다 (경로: insight탭 - fork 항목 클릭)
pull request 탭에 들어가면 풀리퀘스트를 요청받은 항목을 볼 수 있습니다.
file changed 탭을 클릭하면 어떤 새로운 코드가 이 풀 리퀘스트 담겨있는지 확인할 수 있습니다.
변경된 코드의 왼쪽에서 + 버튼을 클릭해서 라인별로 댓글을 달 수 있습니다.
여기에서 수정사항을 제안하거나 질문을 할 수 있습니다. 코드 리뷰를 끝내고 우측 상단의 review changes 버튼을 클릭하면 write 창이 열립니다.
댓글만 달고 싶다면 Comment, 댓글을 달고 코드가 좋아 바로 병합해도 될 거 같으면  Approve, 수정을 요청하고 싶으면 Request changes를 선택합니다.

승인 후 Merge pull request 버튼을 눌러 풀 리퀘스트를 병합합니다. (원본저장소의 소유자만 할 수 있는 작업입니다)

병합 후 insight탭에 contributors 항목에서 컨트리뷰션 활동을 볼 수 있습니다.
컨트리뷰션을 올린 사람은 프로필에서 해당 컨트리뷰션을 등재할 수 있습니다 (자랑용)
