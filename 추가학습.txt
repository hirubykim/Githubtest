개발 현장에서 유용하게 사용하는 5가지 편리한 기능을 적어둡니다.
실수로 파일을 빼먹고 커밋했을 때, 이력을 옛날로 되돌리고 싶을 때 등에 관한 명령어입니다.
이를 익히면 훨씬 더 편하게 버전 관리가 가능합니다.

1. 수정하지 못한 파일이 있어 방금 만든 커밋에 추가하고 싶을 때 : amend 명령어 (원격저장소도 가능함)
방법: 커밋옵션 > "마지막 커밋 정정" 이렇게 하면 지금 올린 변경사항이 기존 커밋에 추가되면서 기존 커밋이 덮어 씌워집니다.
원격저장소에 올라간 커밋도 위의 방법대로 가능하며 마지막에 강제 푸시만 따로 해주면 된다.

2. 다른 브랜치의 커밋 하나만 내 브랜치에 반영하기 : cherry-pick 명령어
만약에 A개발자가 feat/a 브랜치를, B개발자가 feat/b 브랜치를 작업하고 있을 때 B개발자가 A개발자의 커밋 중 하나를 자신의 브랜치에 반영하고 싶다면 어떻게 해야 할까
feat/a 브랜치와 병합을 하게 된다면 나머지 불필요한 커밋도 feat/b 브랜치에 반영이 되니 이런 상황은 원하지 않는다면 cherry-pick 명령어를 사용하는 게 좋다.

방법:
feat/b 브랜치에 있는 상태에서 복제하기 원하는 커밋을 오른쪽 클릭한 후 [체리픽]을 선택합니다. (해당 체리픽 명령어가 성공하면 feat/b 브랜치에 반영됩니다)
체리픽에 대한 안내 메시지가 뜹니다. (특정 커밋의 변경사항만 현재 브랜치에 가져온다는 얘기입니다)

이후 성공적으로 feat/b 브랜치에 원하는 커밋이 붙은 것을 볼 수 있습니다. 

참고: feat/a 브랜치에 복제 당한 커밋과 feat/b 브랜치에 복제된 커밋은 서로 다른 id를 가진 커밋입니다.
변경사항을 복사해 왔지만, 서로 같은 커밋은 아닙니다. 

3. 옛날 커밋으로 브랜치을 되돌리고 싶을 때 : reset 명령어
개발과정에서 이전 사항으로 되돌려달라는 요청을 받은 경우에는 어떻게 해야 할까요?
이럴 때 쓰는 명령어가 reset 명령어 입니다. 중간 중간 커밋을 자주 만들어놔서 옛날 커밋으로 간단히 상태만 되돌리면 됩니다.

방법:
되돌리고 싶은 브랜치의 최신 커밋에서 오른쪽 버튼을 누르고 [이 커밋까지 현재 브랜치를 초기화]를 클릭합니다.
그런데 여기서 3가지 옵션이 있습니다. soft, mixed, hard 입니다.

우리가 보통 하고싶은 깔끔한 리셋은 hard 모드입니다. 지금 작업공간이 어떻게 되어있든 간에 그냥 깔끔하게 히스토리를 돌리겠다는 옵션입니다.
반면 soft 옵션과 mixed 옵션은 원하는 커밋으로 이력을 돌려주긴 하지만 이 다음에 추가했던 모든 변경사항을 작업공간으로 뽑아줍니다.

mixed 옵션을 선택하면 변경사항이 스테이지 아래로 튕겨나온 걸 볼 수 있습니다. 시간은 성공적으로 되돌렸지만 변경사항은 커밋하기 전 상태로 남겨두는 것입니다.
즉, 커밋은 없던 걸로 되돌릴 수 있지만 파일을 만들었던 기록이 스테이지 아래에 살아났으니 수정도 마음대로 가능합니다. 

soft 옵션은 mixed와 유사하지만 차이점이 있습니다. 
mixed 옵션은 변경사항을 스테이지 아래로 둬서 다시 무엇을 스테이지로 add할 지 고민할 수 있지만 soft 옵션은 변경사항을 스테이지 위로 둬서 다시 당장 커밋을 할 수 있다는 차이점이 있습니다.

soft - 모든 로컬 변경사항을 유지
mixed - 작업 상태는 그대로 두지만 인덱스는 초기화
hard - 모든 작업 상태 내 변경사항을 버림
옵션을 선택하고 커밋 초기화를 해줍니다.

hard 옵션의 경우 커밋하지 않은 변경사항을 날릴 수 있어 hard 리셋을 잘못하면 다 날려먹을 수 있습니다. 작업 시 조심해야 합니다.
그리고 hard 옵션으로 되돌리고 나서 원격브랜치에도 반영하고 싶으면 강제 푸시해야 합니다. 

4. 이 커밋의 변경사항을 되돌리고 싶을 때 : revert 명령어
3번에서는 커밋을 마치 없었던 것처럼 되돌리는 방법을 배웠습니다. 
하지만 모두가 함께 쓰는 브랜치라 이력관리가 중요하다면 아무도 모르게 없었던 일처럼 만드는 것보다 변경사항을 되돌리는 새로운 커밋을 만드는 게 더 좋습니다.
이때 필요한 명령어가 revert 명령어 입니다.

방법: 
revert 하고 싶은 커밋을 오른쪽 클릭해서 [커밋 되돌리기]를 선택해줍니다.

명시적으로 커밋을 되돌리는 새로운 커밋이 만들어졌습니다. 해당 revert 명령어는 방금 한 커밋뿐만 아니라 이전에 한 커밋도 얼마든지 되돌릴 수 있습니다. 
잘못된 커밋은 언제든지 revert를 사용해서 되돌리시면 됩니다.

5. 변경사항을 잠시 다른 곳에 저장하고 싶으나 커밋은 안 만들고 싶을 때 : stash 명령어
개발자가 어떤 브랜치에서 개발을 하고 있는 중에 급하게 고쳐야 하는 버그가 생겼습니다. 
다른 브랜치로 이동하려는데 현재 브랜치에 아직 커밋하지 않은 변경사항 파일이 10개 있습니다.
하지만 아직 커밋으로 만들기는 애매해서 해당 변경사항을 잠깐 다른 곳에 넣어두고 이후에 다시 꺼내쓰고 싶을 때 사용하는 명령어가 stash 명령어입니다.

즉, stash 명령어는 커밋하지 않은 변경사항을 서랍 속에 넣어두는 명령어입니다.

방법:
상단 메뉸에 스태시 아이콘을 누르고 해당 스태시에 대한 설명을 적습니다. 
확인 버튼을 누르면 작업 공간이 깔끔해집니다. 이제 변경된 파일에 신경 쓰지 않고 자유롭게 pull이나 브랜치 이동 같은 작업을 할 수 있습니다.

참고: stash에는 tracked 상태 (추적중 - 한번이라도 깃에 올렸던 상태)인 파일들만 들어갑니다. 새로 만든 파일은 untracked 상태이므로 들어가지 않습니다.

스태시를 다시 서랍에서 꺼내려면 왼쪽 사이드 바에 있는 스태시 섹션에서 원하는 스태시를 선택하고 [스태시 적용]을 클릭해주면 됩니다.
스태시를 적용하겠습니까? 라는 메시지 창이 뜨는데 이때 해당 스태시를 계속 사용할 게 아니면 [적용 후 삭제] 체크박스를 선택해줍니다.
스태시에 저장된 변경사항이 다시 작업 공간으로 튀어나는 것을 볼 수 있습니다.
